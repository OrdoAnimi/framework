# Partner Demo Flow

## Purpose

A 20-30 minute demo sequence for consultancy partners to understand the Velocity ecosystem, how it supports sales conversations, and where client data boundaries apply.

## Timing

| Time | Segment | Goal |
|---:|---|---|
| 0-3 min | Problem | Establish decision-flow pain. |
| 3-7 min | Framework | Explain Velocity and four-box model. |
| 7-10 min | Academy | Show learning/certification pathway. |
| 10-14 min | EA tool | Show artefact generation concept. |
| 14-18 min | VAF-SA | Show solution architecture practitioner method. |
| 18-22 min | Engagement offers | Map pain to FastStart / Decision Sprint / PMO alignment. |
| 22-26 min | Sample output | Show executive brief, decision map, artefact register. |
| 26-30 min | Boundaries and next step | Clarify AI/data policy and commercial path. |

## Demo Narrative

1. Most organisations do not have an artefact problem; they have a decision-flow problem.
2. Traditional architecture maps domains. Velocity coordinates domains around decisions.
3. ZenCloud advises. StudioSix produces. Velocity decides.
4. The Academy builds practitioner capability.
5. Tools support drafting and structure but do not replace judgement.
6. Partners can use the model directly, co-branded, or white-labelled.

## Sample Outputs

- executive architecture one-pager
- decision maturity score
- stakeholder decision map
- PMO/EA alignment model
- architecture decision sprint plan
- artefact catalogue
- AI and client data boundary statement

## AI and Data Boundaries

State clearly:

- demos use synthetic data
- confidential data requires approved environment
- AI outputs are drafts
- final artefacts require human review
- partner/client storage controls apply
