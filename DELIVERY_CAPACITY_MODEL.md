# Delivery Capacity Model

## Purpose

This document defines how Phil can operate as a one-person architecture engine without overcommitting, diluting quality, or exposing client data.

## Capacity Principles

- Protect deep architecture thinking time.
- Limit concurrent client-facing engagements.
- Use templates and tools for acceleration, not substitution.
- Keep final judgement human-led.
- Escalate early when delivery demand exceeds available capacity.

## Concurrency Rules

| Work type | Recommended limit |
|---|---:|
| Executive FastStart | 1 active at a time |
| Architecture Decision Sprint | 1 active, 1 preparing |
| Advisory retainer | 2-3 light retainers |
| Partner demo / pre-sales | 2 per week |
| Content / publishing | 1 batch per week |
| Tool/product development | Protected blocks only |

## Subcontractable Tasks

Can be delegated:

- formatting
- slide production
- transcription cleanup
- diagram polishing
- research collection
- template preparation
- website/content publishing support
- non-confidential demo preparation

## Non-Subcontractable Tasks

Do not delegate without explicit trust and controls:

- architecture judgement
- client decision facilitation
- executive recommendations
- confidential client analysis
- final artefact approval
- commercial negotiation
- AI/data boundary decisions
- partner escalation

## Weekly Cadence

Recommended weekly rhythm:

- Monday: portfolio and delivery planning
- Tuesday/Wednesday: client workshops and architecture production
- Thursday: artefact review, partner check-ins, executive packs
- Friday: quality review, content extraction, pipeline updates

## Escalation Triggers

Escalate or pause when:

- more than two major engagements overlap
- quality review time is missing
- client asks for urgent unscheduled output
- partner bypasses agreed workflow
- confidential data path is unclear
- AI output is requested as final advice

## QA Model

Every deliverable must pass:

- decision supported
- audience clear
- assumptions explicit
- risks visible
- traceability maintained
- data boundary checked
- partner/client branding correct
- human review completed
- next action clear

## Capacity Dashboard

Track:

- active engagements
- pending proposals
- delivery hours committed
- review hours protected
- artefacts due this week
- partner dependencies
- blocked decisions
- risk escalations
