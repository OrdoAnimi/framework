# Hermes Intake Routing Matrix

## Purpose

This matrix helps Hermes route requests to the right Velocity offer, artefacts, tools, branding mode, and next action.

| Request type | Symptoms / trigger phrases | Routed offer | Required inputs | Recommended artefacts | Tool / workflow | Branding mode | Next action |
|---|---|---|---|---|---|---|---|
| Executive Architecture FastStart | "We need clarity fast"; "board wants a view"; "architecture risk is unclear" | Executive Architecture FastStart | sponsor, business context, current risks, decision list | one-pager, executive readout, 90-day plan | FastStart pack | ZenCloud / partner | Schedule executive discovery. |
| Architecture Decision Sprint | "Decisions are stuck"; "too many documents"; "governance is slow" | Architecture Decision Sprint | decision backlog, stakeholder list, artefacts, governance forums | decision map, maturity assessment, sprint plan | Decision Sprint | Velocity / partner | Run qualification checklist. |
| VAF-SA Engagement Diagnostic | "Solution architecture is inconsistent"; "design quality varies" | VAF-SA Engagement Diagnostic | solution context, architecture artefacts, delivery constraints | SA diagnostic, guardrails, ADR candidates | VAF-SA | Velocity / partner | Book SA review session. |
| PMO Governance Alignment | "PMO and architecture are disconnected"; "status reports lack decision context" | PMO Governance Alignment | PMO cadence, RAID, decision log, reporting packs | PMO/EA alignment, governance cadence, executive snapshot | PMO Portal | Partner / ZenCloud | Map PMO and EA decision flow. |
| Velocity AI Enablement | "Teams use AI but governance is unclear"; "prompts are uncontrolled" | Velocity AI Enablement | AI use cases, data policy, tool list, risk appetite | AI policy, workflow, review controls | AI/data policy | ZenCloud / StudioSix | Run AI safety discovery. |
| Data Readiness | "Data quality blocks delivery"; "AI readiness unclear" | Data readiness assessment | data domains, systems, ownership, risks | data readiness score, decision list, action plan | Future Velocity Data Framework | ZenCloud | Scope readiness review. |
| Security Readiness | "Security appears late"; "risk controls unclear" | Security readiness assessment | risk register, security controls, architecture context | security readiness score, control map | Future Velocity Security Framework | ZenCloud | Include security architect review. |
| Business Architecture | "Capabilities are unclear"; "strategy does not map to delivery" | Business architecture assessment | capabilities, outcomes, value streams, operating model | capability map, value decision map | Future Velocity Business Architecture | ZenCloud | Run business architecture discovery. |
| Academy / Certification | "We need practitioner capability"; "train our architects" | Academy / certification pathway | audience, skill gaps, target role | learning path, course outline, certification plan | Velocity Academy | Academy / StudioSix | Define cohort and pathway. |
| StudioSix Media / Production | "We need content, training, or publishing assets" | StudioSix Media / Production | audience, topic, format, deadline | article, deck, script, video outline | Publishing pipeline | StudioSix | Create production brief. |
| StudioSix Labs / Trading | "Personal dashboard or experimental workflow" | Labs / trading boundary | objective, data source, risk boundary | lab note, disclaimer, prototype backlog | StudioSix Labs | Internal/personal | Keep separate from client delivery. |

## Routing Principles

- Route decision problems to Velocity.
- Route solution practice issues to VAF-SA.
- Route execution visibility issues to PMO Portal.
- Route artefact production to EA Artefact Generator.
- Route capability development to Academy.
- Route publishing and digital product work to StudioSix.
- Keep trading/labs separate from client architecture delivery.

## Safety Flags

Hermes must flag:

- confidential client data
- public tool misuse
- unsupported AI claims
- missing sponsor
- unclear commercial owner
- white-label requirements
- regulated or security-sensitive context
